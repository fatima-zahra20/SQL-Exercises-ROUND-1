💡 Concept 1 — HAVING vs WHERE

❌ WHERE — filters ROWS before grouping

SELECT Entity, AVG("Cardiovascular diseases")
FROM causes_of_death
WHERE Year > 2000  -- filters rows first
GROUP BY Entity

✅ HAVING — filters GROUPS after grouping

SELECT Entity, AVG("Cardiovascular diseases")
FROM causes_of_death
GROUP BY Entity
HAVING AVG("Cardiovascular diseases") > 50000

⚠️ Column names with spaces need quotes in SQL: "Cardiovascular diseases" not just Cardiovascular diseases

💡 Concept 2 — JOINs with CTEs

-- Create two summaries then JOIN them
WITH summary1 AS (
    SELECT Entity, AVG("Cardiovascular diseases") AS avg_cardio
    FROM causes_of_death
    GROUP BY Entity
),
summary2 AS (
    SELECT Entity, AVG("Malaria") AS avg_malaria
    FROM causes_of_death
    GROUP BY Entity
)
SELECT s1.Entity, s1.avg_cardio, s2.avg_malaria
FROM summary1 s1
JOIN summary2 s2 ON s1.Entity = s2.Entity

Since we only have ONE table, JOINs here work by joining CTEs — summaries you create on the fly and then combine.

💡 Concept 3 — Subqueries

-- Find countries with MORE malaria deaths than the world average
SELECT Entity, "Malaria"
FROM causes_of_death
WHERE Year = 2019
  AND "Malaria" > (
      SELECT AVG("Malaria")   -- runs first → returns one number
      FROM causes_of_death
      WHERE Year = 2019
  )
ORDER BY "Malaria" DESC

The subquery calculates the average first. Then the outer query uses that number to filter. Inner query always runs first.

HAVING
EX 01
Find all countries (Entity) where the average Cardiovascular diseases deaths across all years is above 100,000. Show Entity and average, ordered highest first.
easy HAVING hint → GROUP BY Entity, HAVING AVG("Cardiovascular diseases") > 100000
// answer
query = """
SELECT 
    Entity,
    ROUND(AVG("Cardiovascular diseases"), 0) AS avg_cardio_deaths
FROM causes_of_death
GROUP BY Entity
HAVING AVG("Cardiovascular diseases") > 100000
ORDER BY avg_cardio_deaths DESC
"""
pd.read_sql(query, conn)
💡 Column names with spaces MUST be wrapped in double quotes in SQLite. "Cardiovascular diseases" not Cardiovascular diseases — otherwise SQL thinks it's two separate things.
EX 02
Find countries that appear in MORE than 20 years of data AND have average Malaria deaths above 10,000. Show Entity, year count, and average Malaria deaths.
medium HAVING hint → HAVING COUNT(*) > 20 AND AVG("Malaria") > 10000
// answer
query = """
SELECT 
    Entity,
    COUNT(*) AS years_of_data,
    ROUND(AVG("Malaria"), 0) AS avg_malaria_deaths
FROM causes_of_death
GROUP BY Entity
HAVING COUNT(*) > 20 
   AND AVG("Malaria") > 10000
ORDER BY avg_malaria_deaths DESC
"""
pd.read_sql(query, conn)
💡 COUNT(*) counts rows — in this dataset each row is one year for one country. So COUNT(*) > 20 means we have more than 20 years of data for that country.
EX 03
Filter for Year 2019 only (WHERE), then find countries where BOTH HIV/AIDS deaths AND Tuberculosis deaths average above 5,000. Use HAVING for both conditions.
medium WHERE + HAVING hint → WHERE Year = 2019 first, then GROUP BY Entity, then HAVING both conditions
// answer
query = """
SELECT 
    Entity,
    ROUND(AVG("HIV/AIDS"), 0) AS avg_hiv,
    ROUND(AVG("Tuberculosis"), 0) AS avg_tb
FROM causes_of_death
WHERE Year = 2019
GROUP BY Entity
HAVING AVG("HIV/AIDS") > 5000
   AND AVG("Tuberculosis") > 5000
ORDER BY avg_hiv DESC
"""
pd.read_sql(query, conn)
💡 WHERE Year = 2019 filters to one year per country — so AVG() here is just the value for that year. The real power is HAVING filtering which countries meet BOTH disease thresholds.
JOINs with CTEs
EX 04
Using a CTE for average Cardiovascular deaths per country, JOIN it back to the main table to show each year's data alongside the country's all-time average. Show Entity, Year, that year's deaths, and the country average. Limit to Morocco.
medium JOIN + CTE hint → WITH avg_cardio AS (...) SELECT ... FROM causes_of_death JOIN avg_cardio ON Entity
// answer
query = """
WITH country_avg AS (
    SELECT 
        Entity,
        ROUND(AVG("Cardiovascular diseases"), 0) AS avg_cardio
    FROM causes_of_death
    GROUP BY Entity
)
SELECT 
    c.Entity,
    c.Year,
    c."Cardiovascular diseases" AS this_year_deaths,
    ca.avg_cardio AS country_avg
FROM causes_of_death c
JOIN country_avg ca ON c.Entity = ca.Entity
WHERE c.Entity = 'Morocco'
ORDER BY c.Year
"""
pd.read_sql(query, conn)
💡 This is the SQL version of pandas .transform('mean') — adding the group average back to each row so you can compare individual years to the overall trend. 🇲🇦
EX 05
Using TWO CTEs: one for average Cardiovascular deaths per country, one for average Diabetes deaths per country. JOIN them to show both side by side. Which country has high cardiovascular AND high diabetes deaths?
medium JOIN + 2 CTEs hint → WITH cte1 AS (...), cte2 AS (...) JOIN on Entity
// answer
query = """
WITH cardio_avg AS (
    SELECT Entity,
           ROUND(AVG("Cardiovascular diseases"), 0) AS avg_cardio
    FROM causes_of_death
    GROUP BY Entity
),
diabetes_avg AS (
    SELECT Entity,
           ROUND(AVG("Diabetes mellitus"), 0) AS avg_diabetes
    FROM causes_of_death
    GROUP BY Entity
)
SELECT 
    c.Entity,
    c.avg_cardio,
    d.avg_diabetes
FROM cardio_avg c
JOIN diabetes_avg d ON c.Entity = d.Entity
ORDER BY c.avg_cardio DESC
LIMIT 15
"""
pd.read_sql(query, conn)
💡 Two CTEs separated by a comma — each one calculates one metric. Then JOIN combines them like two puzzle pieces using Entity as the shared key.
EX 06
Create a CTE with total deaths from Conflict and terrorism per country. LEFT JOIN it with a CTE of total Cardiovascular deaths. Show all countries — even those with zero conflict deaths. Order by conflict deaths descending.
hard LEFT JOIN hint → LEFT JOIN keeps ALL rows from left CTE — missing matches show as NULL
// answer
query = """
WITH cardio AS (
    SELECT Entity,
           ROUND(SUM("Cardiovascular diseases"), 0) AS total_cardio
    FROM causes_of_death
    GROUP BY Entity
),
conflict AS (
    SELECT Entity,
           ROUND(SUM("Conflict and terrorism"), 0) AS total_conflict
    FROM causes_of_death
    WHERE "Conflict and terrorism" > 0
    GROUP BY Entity
)
SELECT 
    ca.Entity,
    ca.total_cardio,
    co.total_conflict
FROM cardio ca
LEFT JOIN conflict co ON ca.Entity = co.Entity
ORDER BY co.total_conflict DESC
LIMIT 20
"""
pd.read_sql(query, conn)
💡 LEFT JOIN keeps ALL countries from the cardio CTE — even countries with zero conflict deaths (they show NULL). Use LEFT JOIN when you don't want to lose any records from your main dataset.
Subqueries
EX 07
For year 2019, find all countries where Malaria deaths were above the world average Malaria deaths for that same year. Show Entity and Malaria deaths.
medium Subquery in WHERE hint → WHERE "Malaria" > (SELECT AVG("Malaria") FROM ... WHERE Year = 2019)
// answer
query = """
SELECT 
    Entity,
    "Malaria" AS malaria_deaths
FROM causes_of_death
WHERE Year = 2019
  AND "Malaria" > (
      SELECT AVG("Malaria")
      FROM causes_of_death
      WHERE Year = 2019
  )
ORDER BY malaria_deaths DESC
"""
pd.read_sql(query, conn)
💡 The subquery and outer query both filter for Year = 2019 — this is important! You want to compare each country to the world average for the SAME year, not all years combined.
EX 08
Find the year where global Cardiovascular deaths were the highest. Use a subquery — without just using ORDER BY + LIMIT 1.
medium Subquery in HAVING hint → GROUP BY Year, HAVING SUM(...) = (SELECT MAX(total) FROM (subquery))
// answer
query = """
SELECT 
    Year,
    ROUND(SUM("Cardiovascular diseases"), 0) AS total_cardio
FROM causes_of_death
GROUP BY Year
HAVING SUM("Cardiovascular diseases") = (
    SELECT MAX(total_c)
    FROM (
        SELECT SUM("Cardiovascular diseases") AS total_c
        FROM causes_of_death
        GROUP BY Year
    )
)
"""
pd.read_sql(query, conn)
💡 Nested subquery — the innermost calculates totals per year, the middle finds the max, and HAVING matches the year with that max total. Three levels of logic in one query!
EX 09
BOSS: Find countries where Cardiovascular deaths are above the global average AND Malaria deaths are BELOW the global average. These are likely developed countries — high chronic disease, low infectious disease. Use subqueries for both conditions. Filter for Year 2019.
hard WHERE + 2 Subqueries hint → WHERE "Cardiovascular diseases" > (subquery) AND "Malaria" < (subquery)
// answer
query = """
SELECT 
    Entity,
    ROUND("Cardiovascular diseases", 0) AS cardio_deaths,
    ROUND("Malaria", 0) AS malaria_deaths
FROM causes_of_death
WHERE Year = 2019
  AND "Cardiovascular diseases" > (
      SELECT AVG("Cardiovascular diseases")
      FROM causes_of_death
      WHERE Year = 2019
  )
  AND "Malaria" < (
      SELECT AVG("Malaria")
      FROM causes_of_death
      WHERE Year = 2019
  )
ORDER BY cardio_deaths DESC
LIMIT 20
"""
pd.read_sql(query, conn)
💡 This query tells a real public health story — countries with high cardiovascular but low malaria deaths are typically wealthier nations where infectious diseases are controlled but aging populations face chronic disease. Real insight from real data. 🌍
