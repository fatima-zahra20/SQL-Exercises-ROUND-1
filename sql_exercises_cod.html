EX 01
Show all data for Morocco across all years. Display Entity, Year, Cardiovascular diseases, Malaria, and HIV/AIDS deaths. Order by Year ascending.
easy SELECT + WHERE hint → WHERE Entity = 'Morocco' ORDER BY Year
// answer
query = """
SELECT 
    Entity,
    Year,
    "Cardiovascular diseases",
    "Malaria",
    "HIV/AIDS"
FROM causes_of_death

"""

EX 02
Find all rows where Conflict and terrorism deaths were above 10,000 AND the Year is between 2010 and 2020. Show Entity, Year, and Conflict deaths. Order by deaths descending.
easy WHERE + BETWEEN hint → WHERE "Conflict and terrorism" > 10000 AND Year BETWEEN 2010 AND 2020
// answer
query = """
SELECT 
    Entity,
    Year,
    ROUND("Conflict and terrorism", 0) AS conflict_deaths
FROM causes_of_death
WHERE "Conflict and terrorism" > 10000
  AND Year BETWEEN 2010 AND 2020
ORDER BY conflict_deaths DESC
"""
pd.read_sql(query, conn)
💡 BETWEEN is inclusive — BETWEEN 2010 AND 2020 includes both 2010 and 2020. It's cleaner than writing Year >= 2010 AND Year <= 2020.
GROUP BY + HAVING
EX 03–04
EX 03
For each country (Entity), calculate total Cardiovascular diseases deaths across ALL years. Show only countries with more than 5 million total deaths. Order by total descending.
easy GROUP BY + HAVING hint → SUM("Cardiovascular diseases"), HAVING SUM(...) > 5000000
// answer
query = """
SELECT 
    Entity,
    ROUND(SUM("Cardiovascular diseases"), 0) AS total_cardio_deaths
FROM causes_of_death
GROUP BY Entity
HAVING SUM("Cardiovascular diseases") > 5000000
ORDER BY total_cardio_deaths DESC
"""
pd.read_sql(query, conn)
💡 SUM() adds up all values in a column per group. Here it sums all years of cardiovascular deaths for each country — giving you the historical total burden per nation.
EX 04
For each Year, find the total global Malaria deaths. Only show years where global Malaria deaths exceeded 500,000. Is Malaria increasing or decreasing over time?
medium GROUP BY Year + HAVING hint → GROUP BY Year, SUM("Malaria"), HAVING SUM > 500000, ORDER BY Year
// answer
query = """
SELECT 
    Year,
    ROUND(SUM("Malaria"), 0) AS global_malaria_deaths
FROM causes_of_death
GROUP BY Year
HAVING SUM("Malaria") > 500000
ORDER BY Year ASC
"""
pd.read_sql(query, conn)
💡 Grouping by Year instead of Entity lets you see global trends over time. If the years with 500k+ deaths cluster in the early years — Malaria is declining. If recent years appear — it's still a major problem.
CTEs — WITH clause
EX 05–06
EX 05
Using a CTE, first calculate total deaths from ALL of these causes per country: Malaria + HIV/AIDS + Tuberculosis + Neonatal disorders. Call this 'infectious_total'. Then show the top 10 countries with highest infectious disease burden.
medium CTE hint → WITH infectious AS (SELECT Entity, SUM(col1 + col2 + col3 + col4) ...) SELECT ... FROM infectious
// answer
query = """
WITH infectious AS (
    SELECT 
        Entity,
        ROUND(SUM(
            "Malaria" + 
            "HIV/AIDS" + 
            "Tuberculosis" + 
            "Neonatal disorders"
        ), 0) AS infectious_total
    FROM causes_of_death
    GROUP BY Entity
)
SELECT Entity, infectious_total
FROM infectious
ORDER BY infectious_total DESC
LIMIT 10
"""
pd.read_sql(query, conn)
💡 You can add columns directly in SQL: SUM(col1 + col2 + col3) adds them per row first, then sums across all rows. This creates a composite metric — just like your pandas composite_stress_score!
EX 06
Using TWO CTEs: one for total infectious deaths (Malaria + HIV/AIDS + Tuberculosis) per country, one for total chronic deaths (Cardiovascular diseases + Neoplasms + Diabetes mellitus) per country. JOIN them and find countries where chronic deaths are more than 10x the infectious deaths.
hard 2 CTEs + JOIN + HAVING hint → JOIN on Entity, then HAVING chronic_total > infectious_total * 10
// answer
query = """
WITH infectious AS (
    SELECT Entity,
           ROUND(SUM("Malaria" + "HIV/AIDS" + "Tuberculosis"), 0) AS infectious_total
    FROM causes_of_death
    GROUP BY Entity
),
chronic AS (
    SELECT Entity,
           ROUND(SUM(
               "Cardiovascular diseases" + 
               "Neoplasms" + 
               "Diabetes mellitus"
           ), 0) AS chronic_total
    FROM causes_of_death
    GROUP BY Entity
)
SELECT 
    i.Entity,
    i.infectious_total,
    c.chronic_total,
    ROUND(c.chronic_total * 1.0 / i.infectious_total, 1) AS ratio
FROM infectious i
JOIN chronic c ON i.Entity = c.Entity
WHERE i.infectious_total > 0
  AND c.chronic_total > i.infectious_total * 10
ORDER BY ratio DESC
LIMIT 15
"""
pd.read_sql(query, conn)
💡 Countries where chronic deaths are 10x infectious deaths are typically wealthy nations — they've conquered infectious disease but face aging-related chronic conditions. This is the epidemiological transition in data form. 🌍
Subqueries
EX 07–08
EX 07
For year 2019, find countries where Diabetes mellitus deaths are above the global average for that year. Show Entity, diabetes deaths, and the difference from the global average.
medium Subquery in WHERE + SELECT hint → use subquery in WHERE and again in SELECT to calculate difference
// answer
query = """
SELECT 
    Entity,
    ROUND("Diabetes mellitus", 0) AS diabetes_deaths,
    ROUND("Diabetes mellitus" - (
        SELECT AVG("Diabetes mellitus")
        FROM causes_of_death
        WHERE Year = 2019
    ), 0) AS above_global_avg
FROM causes_of_death
WHERE Year = 2019
  AND "Diabetes mellitus" > (
      SELECT AVG("Diabetes mellitus")
      FROM causes_of_death
      WHERE Year = 2019
  )
ORDER BY diabetes_deaths DESC
LIMIT 15
"""
pd.read_sql(query, conn)
💡 You can use the same subquery in both WHERE (to filter) and SELECT (to calculate difference). SQL runs it each time — or you can use a CTE to calculate it once and reuse it.
EX 08
Find the year with the highest global Self-harm deaths. Then show ALL countries' Self-harm deaths for that specific year, ordered highest first.
medium Subquery in WHERE hint → WHERE Year = (SELECT Year FROM ... GROUP BY Year ORDER BY SUM DESC LIMIT 1)
// answer
query = """
SELECT 
    Entity,
    Year,
    ROUND("Self-harm", 0) AS self_harm_deaths
FROM causes_of_death
WHERE Year = (
    SELECT Year
    FROM causes_of_death
    GROUP BY Year
    ORDER BY SUM("Self-harm") DESC
    LIMIT 1
)
ORDER BY self_harm_deaths DESC
LIMIT 20
"""
pd.read_sql(query, conn)
💡 The subquery finds the peak year for global self-harm deaths. The outer query then shows every country's data for that year. Two separate questions answered in one query.
Window Functions ⭐
EX 09–11
EX 09
For each row, show Entity, Year, Cardiovascular deaths, AND the average Cardiovascular deaths for that country across all years — side by side. Limit to Morocco and show all years.
medium OVER PARTITION BY hint → AVG("Cardiovascular diseases") OVER (PARTITION BY Entity)
// answer
query = """
SELECT 
    Entity,
    Year,
    ROUND("Cardiovascular diseases", 0) AS cardio_deaths,
    ROUND(AVG("Cardiovascular diseases") OVER (
        PARTITION BY Entity
    ), 0) AS country_avg_cardio
FROM causes_of_death
WHERE Entity = 'Morocco'
ORDER BY Year
"""
pd.read_sql(query, conn)
💡 PARTITION BY Entity means "calculate this average separately for each country." Every Morocco row gets Morocco's average — every France row gets France's average. No GROUP BY needed, all rows kept. 🇲🇦
EX 10
Rank countries by their total Cardiovascular deaths in 2019 (rank 1 = highest). Show Entity, deaths, and rank. Then filter to show only the top 10 ranked countries.
medium RANK() + CTE hint → RANK() OVER (ORDER BY "Cardiovascular diseases" DESC), wrap in CTE then filter rank <= 10
// answer
query = """
WITH ranked AS (
    SELECT 
        Entity,
        ROUND("Cardiovascular diseases", 0) AS cardio_deaths,
        RANK() OVER (
            ORDER BY "Cardiovascular diseases" DESC
        ) AS death_rank
    FROM causes_of_death
    WHERE Year = 2019
)
SELECT *
FROM ranked
WHERE death_rank <= 10
"""
pd.read_sql(query, conn)
💡 You can't filter on a window function directly in WHERE — wrap it in a CTE first, then filter. This pattern (CTE + filter on rank) is one of the most common SQL interview questions.
EX 11
For each country, show how Cardiovascular deaths changed year over year using LAG(). Show Entity, Year, current year deaths, previous year deaths, and the difference. Filter for Morocco.
hard LAG() Window Function hint → LAG("Cardiovascular diseases") OVER (PARTITION BY Entity ORDER BY Year)
// answer
query = """
SELECT 
    Entity,
    Year,
    ROUND("Cardiovascular diseases", 0) AS this_year,
    ROUND(LAG("Cardiovascular diseases") OVER (
        PARTITION BY Entity 
        ORDER BY Year
    ), 0) AS prev_year,
    ROUND("Cardiovascular diseases" - LAG("Cardiovascular diseases") OVER (
        PARTITION BY Entity 
        ORDER BY Year
    ), 0) AS year_over_year_change
FROM causes_of_death
WHERE Entity = 'Morocco'
ORDER BY Year
"""
pd.read_sql(query, conn)
💡 LAG() looks at the PREVIOUS row's value. PARTITION BY Entity ensures it only looks at previous years for the same country. Positive change = more deaths. Negative = fewer deaths that year. 🇲🇦
Boss Level
EX 12
EX 12
Full pipeline: (1) Using a CTE, calculate total infectious deaths (Malaria + HIV/AIDS + Tuberculosis) AND total chronic deaths (Cardiovascular + Neoplasms) per country per year. (2) Add a window function to rank countries by infectious deaths within each year. (3) Show only rank 1 countries — the most infectious-disease-burdened country each year. Has the same country dominated every year?
hard CTE + Window + Filter hint → CTE with totals → RANK() OVER (PARTITION BY Year ORDER BY infectious DESC) → filter rank = 1
// answer
query = """
WITH disease_totals AS (
    SELECT 
        Entity,
        Year,
        ROUND("Malaria" + "HIV/AIDS" + "Tuberculosis", 0) AS infectious_total,
        ROUND("Cardiovascular diseases" + "Neoplasms", 0) AS chronic_total
    FROM causes_of_death
    WHERE Code IS NOT NULL  -- filter to actual countries only
),
ranked AS (
    SELECT 
        Entity,
        Year,
        infectious_total,
        chronic_total,
        RANK() OVER (
            PARTITION BY Year 
            ORDER BY infectious_total DESC
        ) AS infectious_rank
    FROM disease_totals
)
SELECT Entity, Year, infectious_total, chronic_total, infectious_rank
FROM ranked
WHERE infectious_rank = 1
ORDER BY Year
"""
pd.read_sql(query, conn)
💡 WHERE Code IS NOT NULL filters out regional aggregates (Africa, World, etc.) keeping only real countries. This is a real data quality step analysts do — and it changes your results significantly. 🌍
