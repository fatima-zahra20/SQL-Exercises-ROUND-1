⚡ Quick Reference — Patterns You Need

WINDOW FUNCTION STRUCTURE

FUNCTION() OVER (
  PARTITION BY col  -- group
  ORDER BY col      -- order within group
)

RANK vs DENSE_RANK vs ROW_NUMBER

1,2,2,4  → RANK (skips after tie)
1,2,2,3  → DENSE_RANK (no skip)
1,2,3,4  → ROW_NUMBER (unique always)

LAG / LEAD

LAG(col, 1, 0)  → previous row
LEAD(col, 1, 0) → next row
-- 1 = rows back/forward
-- 0 = default if NULL

CTE PATTERN

WITH cte1 AS (...),
     cte2 AS (...)
SELECT ...
FROM cte1
JOIN cte2 ON ...

SUBQUERY LOCATIONS

SELECT col - (SELECT AVG...)
WHERE col > (SELECT AVG...)
HAVING SUM() = (SELECT MAX...
  FROM (SELECT SUM... GROUP BY)
)

FILTER ON WINDOW FUNCTION

WITH ranked AS (
  SELECT ..., RANK() OVER (...) AS r
)
SELECT * FROM ranked
WHERE r = 1  -- filter HERE not WHERE above

EX 01
Find all countries where Tuberculosis deaths in 2010 were above the global average Tuberculosis deaths in 2010. Show Entity and deaths.
easySubquery in WHEREhint → WHERE "Tuberculosis" > (SELECT AVG... WHERE Year = 2010)
// answer
query = """
SELECT Entity, ROUND("Tuberculosis", 0) AS tb_deaths

  AND Code NOT LIKE 'OWID%'

      SELECT AVG("Tuberculosis")
      FROM causes_of_death
      WHERE Year = 2010
        AND Code NOT LIKE 'OWID%'
  )
ORDER BY tb_deaths DESC
"""

EX 02
For each country in 2019, show their Malaria deaths AND how many times larger it is than the global average. Label this 'times_above_avg'. Only show countries above average.
mediumSubquery in SELECThint → "Malaria" / (SELECT AVG("Malaria")...) AS times_above_avg
// answer
query = """
SELECT 
    Entity,
    ROUND("Malaria", 0) AS malaria_deaths,
    ROUND("Malaria" * 1.0 / (
        SELECT AVG("Malaria")
        FROM causes_of_death
        WHERE Year = 2019
          AND Code NOT LIKE 'OWID%'
    ), 1) AS times_above_avg
FROM causes_of_death
WHERE Year = 2019
  AND Code NOT LIKE 'OWID%'
  AND "Malaria" > (
      SELECT AVG("Malaria")
      FROM causes_of_death
      WHERE Year = 2019
        AND Code NOT LIKE 'OWID%'
  )
ORDER BY times_above_avg DESC
"""
pd.read_sql(query, conn)
💡 Same subquery used in both SELECT (to calculate ratio) and WHERE (to filter). Remember * 1.0 for decimal division in SQLite.
EX 03
Find the country with the LOWEST Cardiovascular deaths in 2019 (excluding zeros). Use a subquery — not ORDER BY + LIMIT.

SELECT Entity, ROUND("Cardiovascular diseases", 0) AS cardio_deaths

WHERE Year = 2019
  AND Code NOT LIKE 'OWID%'
  AND "Cardiovascular diseases" = (

EX 04
Find years where global Neonatal disorders deaths were BELOW the all-time average across all years. Show Year and total deaths. Is neonatal mortality improving?
mediumSubquery in HAVINGhint → GROUP BY Year, HAVING SUM(...) < (SELECT AVG of yearly totals)
// answer
query = """
SELECT Year, ROUND(SUM("Neonatal disorders"), 0) AS total_neonatal
FROM causes_of_death
WHERE Code NOT LIKE 'OWID%'
GROUP BY Year
HAVING SUM("Neonatal disorders") < (
    SELECT AVG(yearly_total)
    FROM (
        SELECT SUM("Neonatal disorders") AS yearly_total
        FROM causes_of_death
        WHERE Code NOT LIKE 'OWID%'
        GROUP BY Year
    )
)
ORDER BY Year ASC
"""
pd.read_sql(query, conn)
💡 Nested subquery in HAVING — inner calculates yearly totals, middle calculates average of those totals, HAVING filters years below that average. If recent years appear → improvement!
EX 05
For Morocco specifically, find years where Cardiovascular deaths were above Morocco's OWN average. Show Year and deaths. How many years were above average?

SELECT Year, ROUND("Cardiovascular diseases", 0) AS cardio_deaths
FROM causes_of_death

  AND "Cardiovascular diseases" > (

      FROM causes_of_death
      WHERE Entity = 'Morocco'

pd.read_sql(query, conn)

EX 06
Find countries where their 2019 HIV/AIDS deaths were MORE THAN DOUBLE their 2000 HIV/AIDS deaths. Show Entity, 2000 deaths, 2019 deaths, and the ratio.
hardCorrelated Subqueryhint → subquery to get year 2000 value, then compare to year 2019 outer query
// answer
query = """
SELECT 
    a.Entity,
    ROUND(b."HIV/AIDS", 0) AS deaths_2000,
    ROUND(a."HIV/AIDS", 0) AS deaths_2019,
    ROUND(a."HIV/AIDS" * 1.0 / NULLIF(b."HIV/AIDS", 0), 2) AS ratio
FROM causes_of_death a
JOIN causes_of_death b 
    ON a.Entity = b.Entity 
    AND b.Year = 2000
WHERE a.Year = 2019
  AND a.Code NOT LIKE 'OWID%'
  AND a."HIV/AIDS" > b."HIV/AIDS" * 2
ORDER BY ratio DESC
"""
pd.read_sql(query, conn)
💡 Self-JOIN — joining the same table twice with different aliases (a=2019, b=2000). NULLIF(col, 0) prevents division by zero errors. This is how you compare two different years in one query.
CTEs — Think in Steps
EX 07–12
EX 07
Using a CTE, calculate the total deaths from ALL causes per country per year (sum every death column). Then show the top 5 deadliest countries in 2019.
mediumCTEhint → SUM all columns in CTE, then SELECT from CTE WHERE Year = 2019 LIMIT 5
// answer
query = """
WITH total_deaths AS (
    SELECT Entity, Year,
        ROUND(
            "Meningitis" + "Neoplasms" + "Malaria" + 
            "Drowning" + "HIV/AIDS" + "Tuberculosis" + 
            "Neonatal disorders" + "Diarrheal diseases" + 
            "Cardiovascular diseases" + "Diabetes mellitus" + 
            "Chronic respiratory diseases" + "Self-harm" +
            "Lower respiratory infections" + "Nutritional deficiencies"
        , 0) AS all_cause_total
    FROM causes_of_death
    WHERE Code NOT LIKE 'OWID%'
)
SELECT Entity, Year, all_cause_total
FROM total_deaths
WHERE Year = 2019
ORDER BY all_cause_total DESC
LIMIT 5
"""
pd.read_sql(query, conn)
💡 Adding all columns gives you all-cause mortality — a key public health metric. The CTE makes this readable; doing it in a subquery would be nearly unreadable.
EX 08
Using a CTE, find each country's average Malaria deaths across all years. Then in the main query, show only countries whose average is above the MEDIAN of all country averages. (Hint: use the middle value approach)
hardCTE + Nested Subqueryhint → CTE for averages, then HAVING avg > (SELECT AVG from CTE) as approximation of median
// answer
query = """
WITH country_avg AS (
    SELECT Entity,
           ROUND(AVG("Malaria"), 0) AS avg_malaria
    FROM causes_of_death
    WHERE Code NOT LIKE 'OWID%'
    GROUP BY Entity
)
SELECT Entity, avg_malaria
FROM country_avg
WHERE avg_malaria > (
    SELECT AVG(avg_malaria) FROM country_avg
)
ORDER BY avg_malaria DESC
"""
pd.read_sql(query, conn)
💡 You can reference a CTE inside its own subquery! The CTE is defined once and used twice — in FROM and in the subquery. This is one of CTEs' biggest advantages over subqueries.
EX 09
Using THREE CTEs: (1) total Cardiovascular deaths per country, (2) total Malaria deaths per country, (3) total Diabetes deaths per country. JOIN all three and show countries where cardiovascular is the dominant cause (largest of the three).
hard3 CTEs + JOINhint → JOIN all 3 CTEs, then WHERE cardio > malaria AND cardio > diabetes
// answer
query = """
WITH cardio AS (
    SELECT Entity, ROUND(SUM("Cardiovascular diseases"), 0) AS total_cardio
    FROM causes_of_death WHERE Code NOT LIKE 'OWID%'
    GROUP BY Entity
),
malaria AS (
    SELECT Entity, ROUND(SUM("Malaria"), 0) AS total_malaria
    FROM causes_of_death WHERE Code NOT LIKE 'OWID%'
    GROUP BY Entity
),
diabetes AS (
    SELECT Entity, ROUND(SUM("Diabetes mellitus"), 0) AS total_diabetes
    FROM causes_of_death WHERE Code NOT LIKE 'OWID%'
    GROUP BY Entity
)
SELECT c.Entity, c.total_cardio, m.total_malaria, d.total_diabetes
FROM cardio c
JOIN malaria m ON c.Entity = m.Entity
JOIN diabetes d ON c.Entity = d.Entity
WHERE c.total_cardio > m.total_malaria
  AND c.total_cardio > d.total_diabetes
ORDER BY c.total_cardio DESC
LIMIT 15
"""
pd.read_sql(query, conn)
💡 Three CTEs joined in sequence — c JOIN m JOIN d. Each JOIN adds another dataset. WHERE compares columns across the joined CTEs. This is the SQL equivalent of pandas merge on multiple DataFrames.
EX 10
Using CTEs, find Morocco's year with the highest Cardiovascular deaths AND year with the lowest. Show both in one result with a label ('Peak Year' and 'Best Year').
hard2 CTEs + UNIONhint → CTE for max year, CTE for min year, then UNION ALL to combine
// answer
query = """
WITH morocco AS (
    SELECT Year, ROUND("Cardiovascular diseases", 0) AS cardio
    FROM causes_of_death
    WHERE Entity = 'Morocco'
),
peak AS (
    SELECT Year, cardio, 'Peak Year' AS label
    FROM morocco
    WHERE cardio = (SELECT MAX(cardio) FROM morocco)
),
best AS (
    SELECT Year, cardio, 'Best Year' AS label
    FROM morocco
    WHERE cardio = (SELECT MIN(cardio) FROM morocco)
)
SELECT * FROM peak
UNION ALL
SELECT * FROM best
"""
pd.read_sql(query, conn)
💡 UNION ALL combines results from two queries into one table. Both queries must have the same columns. This pattern is useful for combining "best case" and "worst case" into one summary. 🇲🇦
EX 11
Using a CTE, calculate the percentage each cause contributes to total deaths for each country in 2019. Focus on Cardiovascular diseases, Malaria, and HIV/AIDS. Show Entity and percentages.
hardCTE + Percentageshint → CTE calculates total, main query divides each cause by total * 100
// answer
query = """
WITH totals AS (
    SELECT Entity,
           "Cardiovascular diseases" + "Malaria" + "HIV/AIDS" + 
           "Tuberculosis" + "Neonatal disorders" + "Diabetes mellitus" AS total
    FROM causes_of_death
    WHERE Year = 2019 AND Code NOT LIKE 'OWID%'
)
SELECT 
    c.Entity,
    ROUND("Cardiovascular diseases" * 100.0 / t.total, 1) AS cardio_pct,
    ROUND("Malaria" * 100.0 / t.total, 1) AS malaria_pct,
    ROUND("HIV/AIDS" * 100.0 / t.total, 1) AS hiv_pct
FROM causes_of_death c
JOIN totals t ON c.Entity = t.Entity
WHERE c.Year = 2019 AND c.Code NOT LIKE 'OWID%'
ORDER BY cardio_pct DESC
LIMIT 15
"""
pd.read_sql(query, conn)
💡 * 100.0 forces decimal division for percentages. The CTE calculates the denominator (total) once — then the main query uses it for each percentage calculation.
EX 12
Using CTEs, find countries that went from HIGH Malaria burden (above global average) in 2000 to LOW Malaria burden (below global average) in 2019. These are countries that successfully reduced Malaria.
hard2 CTEs + JOINhint → CTE for 2000 high burden countries, CTE for 2019 low burden countries, JOIN to find overlap
// answer
query = """
WITH high_2000 AS (
    SELECT Entity, ROUND("Malaria", 0) AS malaria_2000
    FROM causes_of_death
    WHERE Year = 2000 AND Code NOT LIKE 'OWID%'
      AND "Malaria" > (
          SELECT AVG("Malaria") FROM causes_of_death
          WHERE Year = 2000 AND Code NOT LIKE 'OWID%'
      )
),
low_2019 AS (
    SELECT Entity, ROUND("Malaria", 0) AS malaria_2019
    FROM causes_of_death
    WHERE Year = 2019 AND Code NOT LIKE 'OWID%'
      AND "Malaria" < (
          SELECT AVG("Malaria") FROM causes_of_death
          WHERE Year = 2019 AND Code NOT LIKE 'OWID%'
      )
)
SELECT h.Entity, h.malaria_2000, l.malaria_2019,
       ROUND((h.malaria_2000 - l.malaria_2019) * 100.0 / h.malaria_2000, 1) AS pct_reduction
FROM high_2000 h
JOIN low_2019 l ON h.Entity = l.Entity
ORDER BY pct_reduction DESC
"""
pd.read_sql(query, conn)
💡 This is real public health analysis — finding countries that made the transition from high to low burden. The pct_reduction shows HOW MUCH they improved. Countries with high pct_reduction are global health success stories.
Window Functions — Own Them
EX 13–19
EX 13
For each country, show each year's Cardiovascular deaths AND a running total (cumulative sum) of deaths up to that year. Limit to Morocco.
mediumSUM() OVERhint → SUM("Cardiovascular diseases") OVER (PARTITION BY Entity ORDER BY Year)
// answer
query = """
SELECT 
    Entity, Year,
    ROUND("Cardiovascular diseases", 0) AS annual_deaths,
    ROUND(SUM("Cardiovascular diseases") OVER (
        PARTITION BY Entity 
        ORDER BY Year
    ), 0) AS cumulative_deaths
FROM causes_of_death
WHERE Entity = 'Morocco'
ORDER BY Year
"""
pd.read_sql(query, conn)
💡 SUM() OVER with ORDER BY creates a running total — each row adds to the previous total. Without ORDER BY it would just show the grand total on every row. The ORDER BY inside OVER is what makes it cumulative. 🇲🇦
EX 14
For each year, rank ALL countries by Malaria deaths. Then show only countries ranked 1st, 2nd, and 3rd for year 2010. Use DENSE_RANK instead of RANK.
mediumDENSE_RANKhint → DENSE_RANK() OVER (PARTITION BY Year ORDER BY "Malaria" DESC), filter rank <= 3
// answer
query = """
WITH ranked AS (
    SELECT Entity, Year,
           ROUND("Malaria", 0) AS malaria_deaths,
           DENSE_RANK() OVER (
               PARTITION BY Year 
               ORDER BY "Malaria" DESC
           ) AS malaria_rank
    FROM causes_of_death
    WHERE Code NOT LIKE 'OWID%'
)
SELECT Entity, Year, malaria_deaths, malaria_rank
FROM ranked
WHERE Year = 2010 AND malaria_rank <= 3
ORDER BY malaria_rank
"""
pd.read_sql(query, conn)
💡 DENSE_RANK vs RANK: if two countries tie at rank 2, RANK gives next country rank 4, DENSE_RANK gives rank 3. Use DENSE_RANK when you don't want gaps in ranking numbers.
EX 15
For Morocco, show each year's Cardiovascular deaths alongside the NEXT year's deaths using LEAD(). Also calculate the difference. Which year had the biggest increase coming up?
mediumLEAD()hint → LEAD("Cardiovascular diseases", 1, 0) OVER (PARTITION BY Entity ORDER BY Year)
// answer
query = """
SELECT 
    Entity, Year,
    ROUND("Cardiovascular diseases", 0) AS this_year,
    ROUND(LEAD("Cardiovascular diseases", 1, 0) OVER (
        PARTITION BY Entity ORDER BY Year
    ), 0) AS next_year,
    ROUND(LEAD("Cardiovascular diseases", 1, 0) OVER (
        PARTITION BY Entity ORDER BY Year
    ) - "Cardiovascular diseases", 0) AS upcoming_change
FROM causes_of_death
WHERE Entity = 'Morocco'
ORDER BY Year
"""
pd.read_sql(query, conn)
💡 LEAD() is LAG() in reverse — it looks at the NEXT row instead of the previous one. Useful for forecasting questions: "what's coming next year?" Positive upcoming_change = deaths will increase. 🇲🇦
EX 16
For each country, calculate a 3-year moving average of Cardiovascular deaths. Show Entity, Year, actual deaths, and the moving average. Limit to Morocco.
hardAVG() OVER with ROWShint → AVG() OVER (PARTITION BY Entity ORDER BY Year ROWS BETWEEN 2 PRECEDING AND CURRENT ROW)
// answer
query = """
SELECT 
    Entity, Year,
    ROUND("Cardiovascular diseases", 0) AS actual_deaths,
    ROUND(AVG("Cardiovascular diseases") OVER (
        PARTITION BY Entity 
        ORDER BY Year
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ), 0) AS moving_avg_3yr
FROM causes_of_death
WHERE Entity = 'Morocco'
ORDER BY Year
"""
pd.read_sql(query, conn)
💡 ROWS BETWEEN 2 PRECEDING AND CURRENT ROW means "include this row + 2 rows before it" — creating a 3-year window. Moving averages smooth out yearly fluctuations to show the trend clearly.
EX 17
Using ROW_NUMBER(), assign a unique number to each country within each year based on Tuberculosis deaths (highest = 1). Then show only row_number = 1 for each year from 2010 to 2019. Has the same country been #1 every year?
mediumROW_NUMBER()hint → ROW_NUMBER() OVER (PARTITION BY Year ORDER BY "Tuberculosis" DESC), filter rn = 1
// answer
query = """
WITH numbered AS (
    SELECT Entity, Year,
           ROUND("Tuberculosis", 0) AS tb_deaths,
           ROW_NUMBER() OVER (
               PARTITION BY Year 
               ORDER BY "Tuberculosis" DESC
           ) AS rn
    FROM causes_of_death
    WHERE Code NOT LIKE 'OWID%'
)
SELECT Entity, Year, tb_deaths, rn
FROM numbered
WHERE rn = 1 AND Year BETWEEN 2010 AND 2019
ORDER BY Year
"""
pd.read_sql(query, conn)
💡 ROW_NUMBER() always gives unique numbers — no ties. Even if two countries have identical deaths, one gets 1 and one gets 2 (arbitrary for ties). Use RANK() when ties should share the same rank.
EX 18
For each country, show the year-over-year PERCENTAGE CHANGE in Cardiovascular deaths using LAG(). Show Entity, Year, current deaths, previous deaths, and pct_change. Filter for Morocco and only show years where pct_change was negative (deaths decreased).
hardLAG() + Percentagehint → (current - previous) * 100.0 / previous AS pct_change, wrap in CTE then filter pct_change < 0
// answer
query = """
WITH yoy AS (
    SELECT Entity, Year,
           ROUND("Cardiovascular diseases", 0) AS current_deaths,
           ROUND(LAG("Cardiovascular diseases") OVER (
               PARTITION BY Entity ORDER BY Year
           ), 0) AS prev_deaths,
           ROUND(("Cardiovascular diseases" - LAG("Cardiovascular diseases") OVER (
               PARTITION BY Entity ORDER BY Year
           )) * 100.0 / NULLIF(LAG("Cardiovascular diseases") OVER (
               PARTITION BY Entity ORDER BY Year
           ), 0), 2) AS pct_change
    FROM causes_of_death
    WHERE Entity = 'Morocco'
)
SELECT * FROM yoy
WHERE pct_change < 0
ORDER BY pct_change ASC
"""
pd.read_sql(query, conn)
💡 NULLIF prevents division by zero. Wrapping in CTE lets you filter on pct_change — you can't filter on window function results directly in WHERE. Years with negative pct_change are rare improvements in Morocco's cardiovascular trend. 🇲🇦
EX 19
For year 2019, show each country's Cardiovascular deaths AND what PERCENTILE they fall in (0-100). Use PERCENT_RANK() window function. Which percentile is Morocco in?
hardPERCENT_RANK()hint → PERCENT_RANK() OVER (ORDER BY "Cardiovascular diseases") * 100
// answer
query = """
WITH percentiles AS (
    SELECT Entity,
           ROUND("Cardiovascular diseases", 0) AS cardio_deaths,
           ROUND(PERCENT_RANK() OVER (
               ORDER BY "Cardiovascular diseases"
           ) * 100, 1) AS percentile
    FROM causes_of_death
    WHERE Year = 2019 AND Code NOT LIKE 'OWID%'
)
SELECT *
FROM percentiles
WHERE Entity = 'Morocco'
   OR percentile >= 99
ORDER BY percentile DESC
"""
pd.read_sql(query, conn)
💡 PERCENT_RANK() shows where a country sits relative to all others — 90th percentile means higher than 90% of countries. This is how you contextualize a number within a distribution. Where does Morocco rank? 🇲🇦
Boss Levels — Full Pipeline
EX 20–22
EX 20
Find countries that were consistently in the TOP 10 for Cardiovascular deaths for EVERY year from 2010 to 2019. A country must appear in the top 10 ALL 10 years.
hardCTE + RANK + COUNThint → CTE with RANK per year, filter rank <= 10, then GROUP BY Entity HAVING COUNT(*) = 10
// answer
query = """
WITH ranked AS (
    SELECT Entity, Year,
           RANK() OVER (
               PARTITION BY Year 
               ORDER BY "Cardiovascular diseases" DESC
           ) AS cardio_rank
    FROM causes_of_death
    WHERE Code NOT LIKE 'OWID%'
      AND Year BETWEEN 2010 AND 2019
),
top10_per_year AS (
    SELECT Entity, Year
    FROM ranked
    WHERE cardio_rank <= 10
)
SELECT Entity, COUNT(*) AS years_in_top10
FROM top10_per_year
GROUP BY Entity
HAVING COUNT(*) = 10
ORDER BY Entity
"""
pd.read_sql(query, conn)
💡 Two CTEs chained: first ranks countries per year, second filters top 10. Then GROUP BY + HAVING COUNT(*) = 10 finds countries that appeared all 10 years. This is a "consistent performer" analysis.
EX 21
For each country, find the year where Malaria deaths dropped the MOST compared to the previous year (biggest single-year reduction). Show Entity, Year, the drop amount, and rank countries by biggest drop.
hardLAG + CTE + RANKhint → LAG for year-over-year change, CTE to store it, then RANK by biggest drop per country
// answer
query = """
WITH yoy AS (
    SELECT Entity, Year,
           "Malaria" AS malaria,
           LAG("Malaria") OVER (
               PARTITION BY Entity ORDER BY Year
           ) AS prev_malaria,
           "Malaria" - LAG("Malaria") OVER (
               PARTITION BY Entity ORDER BY Year
           ) AS yoy_change
    FROM causes_of_death
    WHERE Code NOT LIKE 'OWID%'
),
biggest_drop AS (
    SELECT Entity, Year, 
           ROUND(malaria, 0) AS malaria_deaths,
           ROUND(yoy_change, 0) AS annual_change,
           ROW_NUMBER() OVER (
               PARTITION BY Entity 
               ORDER BY yoy_change ASC
           ) AS drop_rank
    FROM yoy
    WHERE yoy_change IS NOT NULL
)
SELECT Entity, Year, malaria_deaths, annual_change
FROM biggest_drop
WHERE drop_rank = 1
  AND annual_change < 0
ORDER BY annual_change ASC
LIMIT 15
"""
pd.read_sql(query, conn)
💡 Two window functions in sequence — LAG() to calculate change, then ROW_NUMBER() to find the biggest drop per country. This is a real epidemiological question: which country made the most dramatic single-year Malaria reduction?
EX 22
FINAL BOSS: For each country, calculate: (1) total infectious deaths, (2) total chronic deaths, (3) rank by infectious burden, (4) rank by chronic burden, (5) identify countries where infectious rank is much higher than chronic rank (infectious disease dominates relative to chronic). Show top 15 most "infectious-dominant" countries.
hardFull Pipelinehint → CTE for totals → window ranks → filter where infectious_rank - chronic_rank is most negative
// answer
query = """
WITH disease_burden AS (
    SELECT Entity,
           ROUND(SUM("Malaria" + "HIV/AIDS" + "Tuberculosis" + 
                     "Neonatal disorders"), 0) AS infectious_total,
           ROUND(SUM("Cardiovascular diseases" + "Neoplasms" + 
                     "Diabetes mellitus"), 0) AS chronic_total
    FROM causes_of_death
    WHERE Code NOT LIKE 'OWID%'
    GROUP BY Entity
),
ranked AS (
    SELECT Entity, infectious_total, chronic_total,
           RANK() OVER (ORDER BY infectious_total DESC) AS infectious_rank,
           RANK() OVER (ORDER BY chronic_total DESC) AS chronic_rank
    FROM disease_burden
)
SELECT 
    Entity,
    infectious_total,
    chronic_total,
    infectious_rank,
    chronic_rank,
    chronic_rank - infectious_rank AS rank_gap
FROM ranked
WHERE infectious_total > 0 AND chronic_total > 0
ORDER BY rank_gap DESC
LIMIT 15
"""
pd.read_sql(query, conn)
💡 rank_gap = chronic_rank - infectious_rank. Large positive gap means: low chronic rank (high burden) but high infectious rank (low burden) → infectious disease dominates. Countries with highest rank_gap are the most infectious-disease-burdened relative to their chronic disease profile. This is real global health analysis. 🌍
